- **Фикс:** исправить либо привязку `plan_service` (не включать AI-юриста в Профи), либо логику отображения shelf (считывать `plan_service`, а не просто `services` каталог).
- **Приоритет:** P1 — создаёт ложные ожидания клиента.

### TD-V13: В БД нет Enterprise-тарифа

### TD-V16: Подписка вне регионализации (xlsx vs БД)
- xlsx модель: цены подписок **одинаковые** для Столиц и Регионов (5000/9000/17000).

- Расхождение означает, что либо xlsx — черновая модель, либо БД ещё не обновлена.
- **Фикс:** продуктовое решение + миграция `plans`.
- **Приоритет:** P1.

### TD-V17: ДомКлик и Яндекс.Недвижимость — готовы технически, не подключены к ЛК
_Статус уточнён Игорем 2026-04-23._

- **Партнёрство + техническая интеграция:** готовы. ДомКлик — 45 ₽/размещение по финмодели; Яндекс.Недвижимость — в инфраструктурной готовности.
- **В основном бэкенде `rspase/project/backend@dev` артефактов интеграции нет** (ни endpoint `/feed/domclick`, ни клиента в `app/`, ни env-переменных). Значит интеграции реализованы либо в отдельном репозитории, либо через partnership-сервис, либо ожидают переноса в основной код.
- **В кабинете ЛК `lk.rspace.pro` этих площадок в списке при публикации объекта нет.** На 2026-04-23 юзер может опубликоваться только на Авито и ЦИАН.

  2. Добавить в UI списка площадок при публикации (фронт `/my/listing/{id}/edit` → модалка публикации).
  3. Настроить отображение статусов/звонков/лидов для этих площадок.
  4. Убрать из external-доки «публикуем на 4 площадках» до релиза — это уже сделано (см. `external/00-about.md`, `external/04-publishing.md`, `13-faq.md`).
- **Приоритет:** P2 (не блокер, но продающее обещание — чем быстрее, тем лучше).
