## Критические баги

### BUG-001: `undefined` вместо числа услуг на странице Услуги
- **URL:** `lk.rspase.ru/my/services`
- **Текст:** _«Ваш уровень подписки содержит **undefined** ключевых услуг из 10 возможных!»_
- **Причина (код):** `app/pages/my/Services/components/Subscription.tsx` (ветка `develop`)  
  Строка: `${subscriptionData?.subscription?.plan?.services?.length}` — нет fallback при `null`/`undefined`  
- **Фикс:** заменить на `${subscriptionData?.subscription?.plan?.services?.length ?? 0}`
- **Приоритет:** P0 — видно каждому пользователю с подпиской

### BUG-002: React Hydration Error #418 на каждой загрузке
- **Консоль:** `Minified React error #418` в `entry.client-IDCyV9zu.js`
- **Причина (код):** `app/root.tsx` — условный рендер Yandex.Metrika скрипта через `!import.meta.env.DEV` + `dangerouslySetInnerHTML` вызывает SSR/CSR mismatch  
- **Также:** `suppressHydrationWarning` на `<html>` маскирует проблему, но не решает  
- **Приоритет:** P1 — может ломать интерактивность, увеличивает время до TTI

## UI/UX баги — ЛК (lk.rspase.ru)

### BUG-003: Кнопка «Создать объект» перекрывает нижнюю навигацию
- **Где:** все страницы с bottom navigation
- **Описание:** зелёная CTA-кнопка «Создать объект» fixed-позиционирована прямо над табами навигации, из-за чего при скролле перекрывает их и затрудняет тап по «ОБЪЕКТЫ», «УСЛУГИ», «ПРОФИЛЬ»
- **Приоритет:** P2

### BUG-004: Тарифная карточка «Ультима» обрезана справа на странице Подписки
- **URL:** `lk.rspase.ru/my/subscription`
- **Описание:** третья карточка тарифа (Ультима) обрезается по правому краю — текст фич нечитаем: «10 публика...», «3 онлайн и...», «Заявка на... безлими...»
- **Причина:** горизонтальный overflow в контейнере тарифов, нет горизонтального скролла или wrap
- **Приоритет:** P2

### BUG-005: Обрезанные названия услуг в карточках
- **URL:** `lk.rspase.ru/my/services`
- **Примеры:** «Проверка собственника (Стол...», «Шаблоны всех договоров от п...»
- **Причина:** фиксированная высота заголовка карточки без `text-overflow: ellipsis` с тултипом или переноса
- **Приоритет:** P3

### BUG-006: Нет цены у ряда услуг
- **URL:** `lk.rspase.ru/my/services`
- **Услуги без цены:** «Продвижение на классифайдах», «Купить новостройку», «Пролонгация страховки», «Онлайн-страховой брокер», «Заявка на ипотеку в любой банк»
- **Вопрос:** если услуга бесплатная — нужно писать «Бесплатно» или «Включено в подписку», а не оставлять пустое место
- **Приоритет:** P3

## UI/UX баги — Админка (admin.rspace.pro)

### BUG-007: Колонка «Статус» обрезана в таблице Объекты
- **URL:** `admin.rspace.pro/console/listings`
- **Описание:** последняя колонка «Статус» (с цветными бейджами) обрезана по правому краю, видны только 2-3 символа
- **Приоритет:** P2

### BUG-008: Фильтр «Тип подписки» не содержит всех вариантов
- **URL:** `admin.rspace.pro/console/users` → Фильтры
- **Описание:** доступны только «Базовый» и «Премиум», но в системе есть уровни: Профи, Премиум, Ультима + типы Базовый/Премиум. Фильтр неполный и может вводить в заблуждение
- **Приоритет:** P2

### BUG-009: Несогласованный формат городов
- **URL:** `admin.rspace.pro/console/users`
- **Примеры:** «Г КРАСНОДАР» (верхний регистр с «Г»), «Сочи» (нормальный), «Тюмень» (нормальный)
- **Причина:** вероятно данные приходят из разных источников без нормализации
- **Приоритет:** P3

### BUG-010: Опечатка «Новоросссийск» (три «с»)
- **URL:** `admin.rspace.pro/console/users` → пользователь Коломиец Артём Сергеевич
- **Город:** «Новоросссийск» вместо «Новороссийск»
- **Причина:** нет валидации/нормализации городов при вводе, либо ошибка в справочнике
- **Приоритет:** P3

### BUG-011: Несогласованное форматирование баланса
- **URL:** `admin.rspace.pro/console/users/{id}` → Платежные данные
- **Описание:** «Баланс: 0 ₽» (без копеек) vs «Баланс вывода: 0,00 ₽» (с копейками) — разный формат для однотипных полей
- **Приоритет:** P4

### BUG-012: Путаница в названиях тарифов
- **URL:** `admin.rspace.pro/console/plans`
- **Описание:** два тарифа «Профи» (ID 12 и 13) — отличаются только типом (Базовый/Премиум) и ценой (4000/5000). Также терминология «Уровень» vs «Тип» перегружена: «Уровень: Профи, Тип: Премиум» — непонятно, какой это тариф. В карточке пользователя отображается как «Демо (Профи) (Базовый)»
- **Приоритет:** P2

## Код-ревью: ключевые находки

### CODE-001: Отсутствие null-safe fallback в шаблонных строках
- **Файл:** `app/pages/my/Services/components/Subscription.tsx`
- **Проблема:** optional chaining без `?? default` в строковой интерполяции → `undefined` в UI
- **Рекомендация:** ввести линтер-правило или обёртку для безопасного отображения чисел

### CODE-002: Hydration mismatch из-за условного SSR-рендеринга
- **Файл:** `app/root.tsx`
- **Проблема:** `!import.meta.env.DEV` меняет SSR-вывод vs CSR → React ошибка #418
- **Рекомендация:** перенести Metrika в `useEffect` или использовать `<Script strategy="afterInteractive" />`

### CODE-003: QueryClient как модульный синглтон
- **Файл:** `app/root.tsx` — `export const queryClient = new QueryClient()`
- **Проблема:** при SSR один QueryClient шарится между запросами разных пользователей — потенциальная утечка данных и кеш-коллизии
- **Рекомендация:** создавать `new QueryClient()` внутри компонента или использовать фабрику
