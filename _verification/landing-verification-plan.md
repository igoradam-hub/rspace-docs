# План верификации лендинга `rspace.pro`

**Дата:** 2026-04-23
**Цель:** сверить фактический UX и тексты лендинга с тем, что написано в документации (`external/00-about.md`, `external/01-tariffs.md`, `external/02-start.md`, `external/16-legal.md`, `external/17-contacts.md`) + найти расхождения, о которых дока молчит.
**Инструмент:** Chrome extension (`mcp__Claude_in_Chrome__*`) — режим read-only на браузере, без автоматических действий с оплатой.

## Точки входа и контекст

- **Основной URL:** `https://rspace.pro/`
- **A/B варианты:** в паспорте упомянуты варианты a/b/c/d (разные hero-копии). Нужно определить, делает ли лендинг редирект / Cookie-split / UTM-branch.
- **Из кода `landing/next`:** Next.js 15 app router, static sitemap.xml, prod Yandex Metrika 104015550, reverse-proxy `/api/ingest` → eu.posthog.com.
- **Источники правды:**
  - `_sources/00-product-passport.md` — позиционирование, тарифы, бренд-идентичность.
  - `_sources/01a-tariffs-quickref.md` — актуальные цены и лимиты.
  - `external/00-about.md` — что такое RSpace (наша формулировка).
  - `external/01-tariffs.md` — тарифная сетка (для сверки с тем, что показывают на лендинге).

## Блоки к проверке

### 1. Above-the-fold (Hero)

**Что ищу:**
- Главный заголовок (H1). Точный текст.
- Подзаголовок.
- Primary-CTA — текст кнопки, куда ведёт (страница регистрации или якорь на форму?).
- Secondary-CTA, если есть (например «Посмотреть демо», «Как это работает»).
- Визуал — скриншот кабинета / иллюстрация / видео.
- Indicator'ы доверия (количество риелторов, сделок, партнёров) — есть ли, какие числа.
- Трек-пиксели — проверить, что Yandex Metrika и PostHog загружаются.

**Почему важно:** Hero-копия выходит в Google snippet и share-preview. В доке про неё ничего не написано, а это дефиниция бренда для внешнего мира.

### 2. Прайсинг / тарифы

**Что ищу:**
- Список тарифов на лендинге (как называются, в каком порядке).
- Цены в рублях для каждого тарифа — **Столица** и **Регионы** отдельно или общая цифра.
- Как визуализирована скидка −20/−25/−30%.
- Пробный период (30 дней бесплатно) — как описан. **КРИТИЧНО: упоминается ли привязка карты или нет.**
- Указание валюты, периодичности (в месяц / в год).
- Toggle «Столица / Регионы» (если есть) или single-view.
- Enterprise-блок (индивидуальный тариф) — какой CTA (форма / почта / телефон).

**Почему важно:** уже нашли один баг про карту на триале. На лендинге может быть **своя версия** текста, не совпадающая с `external/01-tariffs.md` — кто копипастит у кого, неизвестно.

### 3. Feature-блоки (что умеет RSpace)

**Что ищу:**
- Список фич на главной. Порядок и формулировки.
- Интеграции, которые афишируются (Авито, ЦИАН, ДомКлик, CloudPayments, OpenAI и т.д.).
- Упоминание **AI-юриста Грут** — в каком контексте, на каком тарифе.
- Публикации на площадках — сколько портaлов заявлено (3? 4? ДомКлик — реально ли?).
- Услуги с ценами — если показаны, какие именно (юрист на сделку, проверка объекта и т.п.).
- Скриншоты продукта / видео демо.

**Почему важно:** расхождение между «что обещают» и «что есть в коде» — главный источник жалоб. Например, дока говорит «ДомКлик интеграции нет в коде», а лендинг может продавать «публикация на 4 площадках».

### 4. Социальные proof

**Что ищу:**
- Отзывы (имена, должности — реальные или заглушки).
- Кейсы / логотипы клиентов.
- Партнёры (банки для ипотеки, страховщики, юридические компании).
- Количественные метрики («10 000 риелторов»).

**Почему важно:** если это реальные имена/компании — нужно в `external/00-about.md` / `external/17-contacts.md` отразить. Если заглушки — знать что текст placeholder.

### 5. Партнёры (ипотека / страхование)

**Что ищу:**
- Конкретный список банков для ипотеки.
- Конкретный список страховщиков.
- Есть ли ссылки / логотипы, работают ли они.

**Почему важно:** В `external/00-about.md` стоит TBD про страховщиков. Лендинг должен давать ответ.

### 6. FAQ / продажный текст

**Что ищу:**
- Блок частых вопросов на лендинге.
- Ответы — сверить с `external/13-faq.md`. Если они разные — разъехалось.

### 7. Footer

**Что ищу:**
- Юридические документы: оферта, политика, cookies, terms. **URL'ы PDF-файлов** — закрывает TBD в `external/16-legal.md`.
- Реквизиты компании: ООО, ИНН/КПП, юр.адрес. Закрывает TBD в `external/16-legal.md`.
- Email поддержки / телефон — закрывает TBD в `external/17-contacts.md`.
- Соцсети — какие активны. Закрывает TBD в `external/17-contacts.md`.
- Ссылка на Telegram-канал (`@rspace_news` — активен ли).
- Copyright, год.

### 8. Форма регистрации

**Что ищу:**
- Где стоит форма (модалка, отдельная страница, якорь?).
- Поля: телефон → SMS-код → пароль → имя (это из кода).
- CTA-текст кнопки отправки.
- Согласие на обработку ПД — отдельный чекбокс, ссылка на политику?
- Текст disclaimers.

**Почему важно:** регистрация — точка конверсии. Тексты ошибок валидации, поведение при «номер уже занят» — всё это должно быть в `external/02-start.md`, сейчас там описано абстрактно.

### 9. SEO и техническое

**Что ищу:**
- `<title>` страницы, `meta description`.
- `og:title`, `og:description`, `og:image` — как лендинг выглядит в шаре.
- `robots.txt` на `rspace.pro/robots.txt`.
- `sitemap.xml` — то же, что в `public/sitemap.xml` репо, или переопределён.
- `canonical` URL.
- `hreflang` (если есть EN-версия).

### 10. Ссылки, которые должны работать

**Что ищу:**
- Кнопка «Войти» → `lk.rspace.pro/auth/login`.
- Ссылка «Зарегистрироваться» → `lk.rspace.pro/auth/register`.
- Юр-ссылки в футере → PDF / HTML страницы.
- Якорные ссылки внутри лендинга (анкоры) — все рабочие?

### 11. Performance / мобильная версия

**Что ищу (быстрая проверка):**
- Загрузка страницы, видно ли Hero сразу.
- Мобильный layout — вертикальная лента, нет horizontal scroll.
- Lazy-load изображений.

### 12. Трекинг и аналитика

**Что ищу:**
- Yandex Metrika: счётчик `104015550` (prod) или `104317883` (test)?
- PostHog: запросы к `/api/ingest/*` летят на `eu.posthog.com`?
- UTM-параметры сохраняются в localStorage / cookies для передачи в регистрацию?

### 13. A/B варианты

**Что ищу:**
- Попробовать открыть с разными `utm_variant=a/b/c/d` → видно ли разные Hero-копии?
- Есть ли Cookie, которая фиксирует вариант для сессии?
- В `RSpace_Landing_Copy_v2.docx` (docx в rspace/) были варианты текста — какие активны сейчас.

```

- CTA: "..." → <URL>

![hero](./screenshots/landing-hero.png)  (если смогу получить)
### Расхождения с докой
- В `external/00-about.md` написано X, на лендинге Y. Нужно обновить X.
### Новые факты для tech-debt / known-issues
- ...
```

По каждому блоку:
1. **Что на экране** — reproducible текст, URL'ы, кнопки.
2. **Скриншот/выжимка DOM** через Chrome-tools.
3. **Расхождение с докой** — точный patch.
4. **Открытые вопросы** — что не удалось проверить (требует логина в ЛК, например — перенесём в волну «ЛК»).

После walkthrough:
1. Обновляю relevant external/ файлы.
2. Заношу новые правки в `CHANGELOG.md` как «Волна 6a (лендинг)».
3. Открытые вопросы (ЛК / админка) переходят в последующие волны.

## Ограничения

- **Не регистрируюсь** — не оставляю реальный номер, не привязываю карту. Триал-флоу смотрю только до экрана «введите SMS-код», дальше — по твоему указанию.
- **Не заполняю формы обратной связи** — только читаю.
- **Все клики по внешним ссылкам** (партнёры, соцсети) — только проверяю что URL валидный, в саму внешнюю страницу не переходим, чтобы не засветить телеметрией чужие сайты.
- **Если лендинг захочет captcha или rate-limit меня** — останавливаюсь и говорю тебе.
